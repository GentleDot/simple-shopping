개요(Overview)

HTTP 동사(HTTP verbs)

본 REST API에서 사용하는 HTTP 동사(verbs)는 가능한한 표준 HTTP와 REST 규약을 따릅니다.

동사(Verb) 용례 (Usage)

GET

리소스를 가져올 때 사용

POST

새 리소스를 만들 때 사용

PUT

기존 리소스를 수정할 때 사용

PATCH

기존 리소스의 일부를 수정할 때 사용

DELETE

기존 리소스를 삭제할 떄 사용

HTTP 상태 코드(HTTP status codes)

본 REST API에서 사용하는 HTTP 상태 코드는 가능한한 표준 HTTP와 REST 규약을 따릅니다.

상태 코드(Status code) 용례(Usage)

200 OK

요청을 성공적으로 처리함

400 BAD_REQUEST

잘못된 요청을 보낸 경우. 응답 본문에 오류에 대한 정보가 담겨있다.

401 UNAUTHORIZED

요구되는 인증 정보가 없어 접근이 거부된 경우.

404 NOT_FOUND

요청한 리소스가 없음.

오류(Errors)

추후 구현될 오류 응답은 오류가 발생했을 때 (상태 코드 >= 400), 본문에 해당 문제를 기술한 JSON 객체가 담겨있다.

예를 들어, 잘못된 요청으로 이벤트를 만들려고 했을 때 400 Bad Request 응답을 받는다.

{"processSuccess" : false, "response" : null, "error" : {"errorMessage" : "about error message", "status" : 400}}

리소스(Resource)

사용자 로그인(Login)

인증이 필요한 요청은 로그인 요청을 통해 인증 토큰을 발급받고 요청 header에 토큰을 싣어야 한다.

로그인 요청(Login Request)

POST 요청을 사용하여 사용자 로그인을 요청한다.

Curl request

$ curl 'http://localhost:8443/login' -i -X POST \
    -H 'Content-Type: application/json;charset=UTF-8' \
    -H 'Accept: application/json' \
    -d '{"username" : "gentledot.wp@gmail.com", "password" : "pass"}'

HTTP request

POST /login HTTP/1.1
Content-Type: application/json;charset=UTF-8
Accept: application/json
Content-Length: 60
Host: localhost:8443

{"username" : "gentledot.wp@gmail.com", "password" : "pass"}
